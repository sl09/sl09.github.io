Hugo搭建博客使用指南

下载Hugo

进入Hugo下载地址：https://github.com/gohugoio/hugo/releases

这里直接下载Windows版本：hugo_xxx_Windows-64bit.zip，下载完成后进行解压，配置环境变量，以便在任意目录下执行hugo命令。

进入命令行界面，这里使用VS Code终端执行命令行操作。

执行如下命令，有正确提示则说明安装成功：

1

hugo version

使用Hugo

执行如下命令完成站点创建：

1

hugo new site sl09Blog

此时已创建了sl09Blog这个目录，该目录即为博客站点，进入该目录。

接下来需要下载并设置主题，进入hugo官网的主题库https://themes.gohugo.io/选择一款主题即可。

选择好主题后，执行git命令将主题从GitHub上clone到本地站点目录：

1

git clone https://github.com/flysnow-org/maupassant-hugo themes/maupassant

完成clone后，继续执行如下命令启动博客：

1

hugo server -t maupassant --buildDrafts

其中-t参数指定主题，--buildDrafts参数的意思是渲染所有的post包括 draft=true的。

启动后在浏览器输入http://localhost:1313/即可本地预览生成的站点，可以通过编辑config.toml文件自定义站点的相关配置。

使用如下命令创建一篇文章：

1

hugo new post/singleton.md

创建完成后会在/content/post目录下生成文件，使用相应工具编辑文件即可。

将个人博客上传的远程仓库

在GitHub上创建名为xxx.github.io的远程仓库，其中xxx为自己的GitHub昵称且为全小写。

在本地建好的站点目录下执行如下命令部署站点：

1

hugo --buildDrafts

执行成功后会在当前目录下生成一个public目录，此目录即为最终部署好的站点。

接下来需要将public目录上传至刚才建好的GitHub远程仓库中。

进入public目录，使用如下命令将此目录初始化为git仓库并提交：

1
2
3

git init
git add .
git commit -m "生成博客并提交"

再次使用如下命令将本地仓库的内容推送到远程仓库：

1
2

git remote add origin https://github.com/sl09/sl09.github.io.git
git push --set-upstream origin master

推送成功后浏览器访问https://sl09.github.io/即可看到部署好的博客站点。
